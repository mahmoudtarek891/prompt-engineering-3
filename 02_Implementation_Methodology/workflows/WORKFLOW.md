# Development Workflow

## Task Lifecycle
1. **Intake**: Task received and registered
2. **Planning**: Estimate, prioritize, assign
3. **Implementation**: Code, test, review
4. **Verification**: QA, integration testing
5. **Deployment**: Release, monitor

## Git Workflow
- `main`: Production-ready code
- `develop`: Integration branch
- `feature/*`: New features
- `fix/*`: Bug fixes

## Review Process
1. Self-review checklist
2. Automated tests pass
3. Peer review
4. Merge approval
